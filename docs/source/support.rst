Действия службы технической поддержки
=====================================

Настоящий раздел описывает типовые действия службы технической поддержки при эксплуатации услуги **V2TextMail**, включая обработку обращений абонентов, 
выполнение операций по управлению услугой и первичную диагностику возникающих проблем.

Типовые обращения абонентов
---------------------------

В рамках эксплуатации услуги **V2TextMail** в службу технической поддержки могут поступать обращения абонентов, связанные с работой сервиса. 
Наиболее распространенными являются следующие категории обращений:

•	отсутствие электронного письма с текстовой версией голосового сообщения;
•	запрос на отключение услуги;
•	запрос на повторное подключение услуги;
•	уточнение принципов работы услуги.

Отключение услуги для абонента
------------------------------

В период эксплуатации услуги **V2TextMail** отключение услуги для абонента осуществляется службой технической поддержки.

Для отключения услуги необходимо:

1. Идентифицировать абонента по номеру :term:`MSISDN`.
2. Проверить наличие записи абонента в базе данных системы **VoiceBox**.
3. Изменить статус активности услуги для соответствующего абонента.

После изменения статуса услуга прекращает применяться к переадресованным вызовам данного абонента. Прямые вызовы и базовая функциональность системы **VoiceBox**
при этом не затрагиваются.

Повторное подключение услуги
----------------------------

Повторное подключение услуги **V2TextMail** для абонента выполняется службой технической поддержки по аналогичной процедуре изменения статуса активности услуги в системе.

Повторное подключение услуги выполняется в соответствии с действующими условиями эксплуатации сервиса и может отличаться от штатной эксплуатации.

Реакция на обращение об отсутствии электронного письма
------------------------------------------------------
В случае обращения абонента с жалобой на отсутствие электронного письма с текстовой версией голосового сообщения службе технической поддержки рекомендуется:

* проверить корректность обработки вызова в системе **VoiceBox**;
* убедиться в наличии голосового сообщения;
* проверить наличие ошибок передачи данных в почтовую систему оператора;
* при необходимости проинформировать абонента о возможных задержках доставки, не зависящих от системы **VoiceBox**.

Ограничения ответственности службы технической поддержки
--------------------------------------------------------

Служба технической поддержки не выполняет администрирование почтовой системы оператора и не несет ответственности за доставку электронных писем после успешной передачи
данных в почтовую систему оператора.

В случае подтвержденной передачи данных посредством интеграционного интерфейса **EmailAPI** дальнейшая обработка и доставка сообщения находятся вне зоны 
ответственности системы **VoiceBox**.
